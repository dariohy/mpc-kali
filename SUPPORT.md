# Support

## Usage questions

Before opening an issue:

1. Read the [README](README.md), [user manual](docs/USER_MANUAL.md), and
   [troubleshooting section](docs/USER_MANUAL.md#19-troubleshooting).
2. Confirm the behavior with the latest stable release.
3. Remove credentials, real targets, customer data, and scanner evidence from
   every command and log excerpt.

Use a GitHub issue for reproducible bugs and scoped feature requests. This is a
community project; response times and implementation are not guaranteed.

## Security vulnerabilities

Do not report vulnerabilities in a public issue. Follow
[SECURITY.md](SECURITY.md) and use a
[private security advisory](https://github.com/dariohy/mcp-kali/security/advisories/new).

## Pentesting engagements

The project cannot authorize testing, validate your scope, operate tools on your
behalf, or provide emergency incident-response services. You are responsible for
written authorization, target scope, tool legality, evidence handling, and
operational safeguards.
